EH2-HT50접속 설정

1. 포트 설정

PIM PORT 6575
EH2 PORT 6575
⚠️ 기본값 사용 가능. 비표준 포트(6570, 9014) 사용 금지 - 헤더 파싱 오류 위험

2. 기기 식별 설정

System Type 42 or 43 or 3

43은 툴스넷8 오픈프로토콜 사양서상, 에스틱. 42는 Focus61용이지만 경험상 성공확률 높음. 43 실패 시 42 or 3 시도

System Number 90

연번이나 연결 테스트라면 데이터 충돌 방지를 위해 큰 값 입력.
System Type 내에서 기기를 고유하게 식별하는 4자리 번호

Station Number 90

특정 작업 위치(스테이션)를 고유하게 식별하는 번호

⚠️ System / Station Number 중복 시 데이터 덮어쓰기 및 리포트 병합 오류 발생

3. MID 통신 순서

연결 수립 절차

  • • MID 0001 Communication Start (통신 시작점)
  • • MID 0012 Application Data Upload Request (설정 동기화)
  • • MID 0008 Data Subscription (데이터 구독)

세션 유지

  • • MID 9999 Keep-alive (고스트 세션 방지, Error 96 예방)

주요 에러 코드

  • • Error 16 - Protocol Busy (컨트롤러 부하, 재전송 최적화 필요)
  • • Error 75 - MID Unsupported (소프트웨어 버전 확인 필요)
  • • Error 96 - Client Already Connected (Keep-alive로 예방)

툴스넷상 보이는 구조

계층 구조

  • • System Type + Number → 기기 유형 식별
  • • Unit (Station) Number → 물리적 장치
  • • Spindle Number → 체결 축
  • • Program Number → 파라미터 세트
v3.7 신규: 단일 소켓 내 다수 Unit 연결 가능. 여러 스테이션을 관리하는 하나의 마스터 컨트롤러나 게이트웨이가 대표로 소켓 하나를 열고, 하위 유닛들의 데이터를 모아서 서버로 던져주는 형태.

PIM 핸드셰이크

Port: 6575

역할: 서버 IP/Port, 최신 시퀀스 번호 제공

MID 12 PIM Verification
MID 07 PIM Info Request
MID 08 PIM Information (IP/Port/Sequence 획득)
3

MID 04 결과 전문

필수: 모든 전문은 4바이트 ASCII 길이로 시작
• Telegram Length: 4 Byte
• Telegram ID: '04'
• Identifier: 5 Byte (ACK 매칭용)
• System Type: '0003' (Generic)
• Rundown Seq No: 순차 증가
• VIN: 25 Byte (차량/바코드)

스핀들 상태값

Overall Status: 0=OK 1=NOK
Torque/Angle: 0=OK 1=NOK 2=Low 3=High
4

세션 유지

Keep-Alive

MID 06, 30초 주기 송신

서버 타임아웃

60초 무응답 시 강제 종료

재연결 로직

  • • Transmission Timeout: 5초 내 ACK 없으면 2회 재전송
  • • 내부 버퍼링: 단절 시 메모리 저장 → 복구 시 순차 재전송
  • • Connection Timeout: 60초 대기 후 재시도
